## üîê Security Best Practices

1. **Never commit secrets** - Use Settings ‚Üí Secrets UI
2. **Rotate secrets regularly** - Update API keys periodically
3. **Enable SSL** - Always use HTTPS in production
4. **Validate user input** - Server-side validation on all tRPC procedures
5. **Rate limit webhooks** - Prevent abuse on Discord/Notion integrations

## üìö API Reference

### WebSocket Connection
```typescript
const ws = new WebSocket('wss://helix-unified-production.up.railway.app/ws');

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  // message.type: 'ucf_update' | 'agent_change' | 'ritual_complete' | 'alert'
  // message.data: { harmony, resilience, prana, drishti, klesha, zoom, ... }
};
```

### tRPC Procedures

**Create Fractal:**
```typescript
trpc.fractals.create.mutate({
  title: "My Fractal",
  description: "Generated from harmony peak",
  ucfState: { harmony: 0.95, resilience: 0.8, ... },
  imageUrl: "data:image/png;base64,...",
  public: true,
});
```

**Fetch User Fractals:**
```typescript
const fractals = await trpc.fractals.listByUser.query();
```

**Favorite Fractal:**
```typescript
await trpc.fractals.toggleFavorite.mutate({ fractalId: "123" });
```

## üêõ Troubleshooting

### WebSocket Connection Fails
**Problem:** "Failed to connect to Helix backend"  
**Solution:** Verify `VITE_HELIX_API_URL` is correct and backend is running

### Fractals Not Saving
**Problem:** "Error saving fractal to database"  
**Solution:** Check user is authenticated and database is accessible

### Portal Links Don't Work
**Problem:** "Portal returns 404"  
**Solution:** Verify portal URLs in `PortalAggregator.tsx` are current
